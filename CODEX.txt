## Forma de trabajo

Cada fase sigue este formato:

1. Codex analiza el estado, riesgos y dependencias.
2. Codex redacta un prompt técnico completo para Claude Code.
3. Claude Code audita el repositorio antes de modificar.
4. Implementa el cambio más pequeño compatible con la arquitectura existente.
5. Ejecuta typecheck, lint, tests, E2E y regresiones.
6. Realiza review adversarial cuando corresponde.
7. Commita y pushea, pero no despliega sin aprobación explícita.
8. El usuario pega en Codex el reporte completo de Claude Code.
9. Codex revisa resultados, detecta vacíos y prepara el deploy o la siguiente fase.
10. Los despliegues se verifican con smoke técnico y, cuando corresponde, una prueba real controlada.

Los reportes deben incluir:

- Qué se auditó.
- Causa raíz.
- Cambios realizados.
- Archivos o módulos afectados.
- Tests y regresiones.
- Commit.
- Estado de deploy.
- Riesgos o pendientes.
- Próximo prompt recomendado.

## Relación Codex–Claude Code

Codex y Claude Code no comparten automáticamente sus conversaciones.

- **Codex:** mantiene la visión global, revisa resultados, define fases, seguridad, alcance y prompts.
- **Claude Code:** trabaja directamente sobre el repositorio, implementa, prueba, commitea y despliega cuando se autoriza.
- **Usuario:** actúa como puente, copiando el prompt de Codex a Claude Code y pegando su reporte nuevamente en Codex.

Codex también puede trabajar directamente en el repositorio si está disponible localmente; Claude Code no es obligatorio. El flujo actual lo utiliza como ejecutor principal y a Codex como coordinador técnico y revisor.
